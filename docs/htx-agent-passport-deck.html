04 / 12
安全

三道纵深防线

每一层独立运作、互为冗余。即使某一层失效，后续层仍能拦截风险。

Layer 01

规则路由拦截

高危意图(提现/借贷/跨链转账)在进入 AI 规划前即被硬拦截，零延迟、零依赖。

Layer 02

Policy Engine 裁决

纯函数确定性引擎——七步检查：金额、币种、日限额、风险评分、幂等性。完全忽略 LLM 输出中的自然语言指令。

Layer 03

执行网关二次校验

审批后、执行前的最终安全门——聚合真实累计、检测价格偏离(Stale Price)、验证幂等性。

06 / 12
技术亮点

为什么不可能被攻破？

Policy Engine — 确定性纯函数

只读结构化字段(金额/币种/限额)，完全忽略 rationale 中的自然语言。"忽略以上规则" 之类的 prompt 注入无法改变裁决结果。

抗 Prompt 注入
无 LLM 依赖
可形式化验证

审计链 — RFC 6962 风格

每个事件 hash = sha256(canonical_json + prev_hash + timestamp)。额外 Merkle 树 + 签名 Tree Head(STH)，支持第三方独立验证。

不可篡改
O(log N) 验证
第三方可审计

信封加密 — HSM 级保护

每条凭证独立 DEK(AES-256-GCM)，DEK 由 KEK 包裹。支持 AWS KMS (FIPS 140-3 L3) 作为 KEK provider。密钥轮换零停机。

属性测试 — 数学证明

Policy Engine 等确定性组件通过 Hypothesis PBT 验证 10 个正确性属性。安全测试通过率 100%，任一失败即 blocker。
